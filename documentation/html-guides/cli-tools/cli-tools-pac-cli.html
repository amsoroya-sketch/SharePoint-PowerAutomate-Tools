PAC CLI (Power Platform CLI)

Power Platform CLI for solution management, environment operations, and deployment.

Installation

Option 1: Using winget (Recommended)

winget install Microsoft.PowerPlatformCLI

Option 2: Using dotnet

dotnet tool install --global Microsoft.PowerApps.CLI.Tool

Option 3: Direct Download

Download from: https://aka.ms/PowerAppsCLI

Verify Installation

pac help
pac --version

Authentication

Create Auth Profile

# Interactive login (opens browser)
pac auth create



With specific environment URL

pac auth create --url https://org.crm.dynamics.com

List Auth Profiles

pac auth list

Output: 

Index Active Kind      Name                    User                      Cloud   Type   Url
[1]   *      DATAVERSE Power Platform Default  user@tenant.onmicrosoft.com PROD  ADMIN  https://org.crm.dynamics.com/

Select Auth Profile

pac auth select --index 1

Check Current Connection

pac env who

Clear Auth

pac auth clear

Environment Commands

List Environments

pac env list

Select Environment

# By name
pac env select --environment "Development"



By ID

pac env select --environment 1a81be40-03fb-e3fe-8964-ac88c74859cc

Get Environment Details

pac env who

Solution Commands

List Solutions

pac solution list

Export Solution

# Unmanaged (development)
pac solution export --name SharePointPermissionScanner --path solution.zip --overwrite



Managed (production)

pac solution export --name SharePointPermissionScanner --path solution_managed.zip --managed --overwrite

Import Solution

# Standard import
pac solution import --path solution.zip



With async (for large solutions)

pac solution import --path solution.zip --async



Force overwrite

pac solution import --path solution.zip --force-overwrite

Clone Solution (for source control)

pac solution clone --name SharePointPermissionScanner --outputDirectory ./src

Pack Solution (from source)

pac solution pack --zipfile solution.zip --folder ./src --packagetype Unmanaged

Application/Component Commands

List Canvas Apps

pac canvas list

Download Canvas App

pac canvas download --canvas-name "AppName" --file app.msapp

Upload Canvas App

pac canvas upload --file app.msapp --canvas-name "AppName"

Common Workflows

Export -> Modify -> Import Cycle

# 1. Export solution
pac solution export --name MySolution --path MySolution.zip --overwrite



2. Extract for editing

Expand-Archive MySolution.zip -DestinationPath MySolution_extracted -Force



3. Make modifications to extracted files


... edit files ...


4. Repack solution

pac solution pack --zipfile MySolution_modified.zip --folder MySolution_extracted



5. Import modified solution

pac solution import --path MySolution_modified.zip --force-overwrite

Source Control Workflow

# Clone solution to folder
pac solution clone --name MySolution --outputDirectory ./src



After changes, pack for deployment

pac solution pack --zipfile deploy.zip --folder ./src --packagetype Unmanaged



Import to target environment

pac solution import --path deploy.zip

Troubleshooting

ErrorSolution
"Not authenticated"Run pac auth create
"Solution not found"Use exact name from pac solution list
"Access denied"Check permissions in Power Platform admin
"Import failed"Check solution dependencies
"Version conflict"Use --force-overwrite flag

Check Solution Import Status

# If async import, check status
pac solution import-status --import-job-id <job-id>

Debug Export Issues

# Export with verbose logging
pac solution export --name MySolution --path MySolution.zip --include all

Useful Flags

FlagDescription
--overwriteOverwrite existing file
--force-overwriteForce import even with conflicts
--asyncRun operation asynchronously
--managedExport as managed solution
--include allInclude all solution components
