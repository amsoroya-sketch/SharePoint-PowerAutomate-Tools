MCP Servers Configuration

Model Context Protocol (MCP) servers provide Claude with real-time access to external systems.

Configuration File

Location: Z:/.mcp.json 

{
  "mcpServers": {
    "sharepoint": {
      "command": "npx",
      "args": ["-y", "@alliottech/sharepoint-mcp-server"],
      "env": {
        "SHAREPOINT_URL": "${SHAREPOINT_URL}",
        "TENANT_ID": "${SHAREPOINT_TENANT_ID}",
        "CLIENT_ID": "${SHAREPOINT_CLIENT_ID}",
        "CLIENT_SECRET": "${SHAREPOINT_CLIENT_SECRET}"
      }
    },
    "azure-devops": {
      "command": "npx",
      "args": ["-y", "@tiberriver256/mcp-server-azure-devops"],
      "env": {
        "AZURE_DEVOPS_ORG": "${AZURE_DEVOPS_ORG}",
        "AZURE_DEVOPS_PAT": "${AZURE_DEVOPS_PAT}"
      }
    },
    "azure": {
      "command": "npx",
      "args": ["-y", "@azure/mcp@latest", "server", "start"],
      "env": {
        "AZURE_SUBSCRIPTION_ID": "${AZURE_SUBSCRIPTION_ID}"
      }
    }
  }
}

Environment Variables

Create a .env file or set system environment variables: 

# SharePoint
SHAREPOINT_URL=https://abctest179.sharepoint.com/sites/Permission-Scanner-Test
SHAREPOINT_TENANT_ID=your-tenant-id
SHAREPOINT_CLIENT_ID=your-client-id
SHAREPOINT_CLIENT_SECRET=your-client-secret



Azure DevOps

AZURE_DEVOPS_ORG=your-organization
AZURE_DEVOPS_PAT=your-personal-access-token



Azure

AZURE_SUBSCRIPTION_ID=your-subscription-id



SQL Server

SQL_SERVER=your-server.database.windows.net
SQL_DATABASE=your-database

Server Details

SharePoint MCP

Package: @alliottech/sharepoint-mcp-server

Capabilities:

  • Search files and documents
  • List SharePoint sites
  • Get site information
  • List document libraries
  • Browse folder contents
  • Read file content
Available Tools:
ToolDescription
search_filesSearch for files in SharePoint
list_sitesList accessible SharePoint sites
get_site_infoGet site details
list_site_drivesList document libraries
list_drive_itemsList files/folders
get_file_contentRead file content

Setup: 1. Create Azure AD app registration 2. Add SharePoint API permissions 3. Grant admin consent 4. Set environment variables

Azure DevOps MCP

Package: @tiberriver256/mcp-server-azure-devops

Capabilities:

  • List projects and repos
  • View pipeline status
  • Access work items
  • Query build results
Setup: 1. Create Personal Access Token (PAT) in Azure DevOps 2. Set AZURE_DEVOPS_ORG and AZURE_DEVOPS_PAT

Create PAT: 1. Go to Azure DevOps > User Settings > Personal Access Tokens 2. Click + New Token 3. Set scopes: Code (Read), Build (Read), Work Items (Read) 4. Copy token immediately

Azure MCP

Package: @azure/mcp

Capabilities:

  • List Azure resources
  • Manage resource groups
  • Query resource properties
  • Execute Azure operations
Setup: 1. Run az login to authenticate 2. Set AZURE_SUBSCRIPTION_ID

MSSQL MCP

Package: @knight0zh/mssql-mcp-server

Capabilities:

  • Execute SQL queries
  • List tables
  • Query data
Setup: 1. Set MSSQL_HOST and MSSQL_DATABASE 2. Configure authentication (Windows/SQL)

Adding New MCP Servers

1. Find Server Package

Browse: https://github.com/modelcontextprotocol/servers

2. Add to .mcp.json

{
  "mcpServers": {
    "new-server": {
      "command": "npx",
      "args": ["-y", "@package/mcp-server"],
      "env": {
        "API_KEY": "${API_KEY}"
      }
    }
  }
}

3. Set Environment Variables

export API_KEY=your-key

4. Restart Claude Code

MCP servers load on startup.

Troubleshooting

Server Not Loading

# Test server manually
npx -y @alliottech/sharepoint-mcp-server

Authentication Errors

  • Verify environment variables are set
  • Check credentials are valid
  • Ensure API permissions granted

Connection Timeout

  • Check network connectivity
  • Verify URLs are correct
  • Check firewall rules

View Logs

MCP servers output to stderr - check terminal for errors.

  • Store secrets in environment variables, not in config
  • Use Azure Key Vault for production secrets

  • Limit API permissions to minimum required
