Azure DevOps MCP Server

Real-time Azure DevOps access for Claude via Model Context Protocol.

Overview

Package: @tiberriver256/mcp-server-azure-devops Repository: https://github.com/tiberriver256/mcp-server-azure-devops

Configuration

.mcp.json Entry

{
  "azure-devops": {
    "command": "npx",
    "args": ["-y", "@tiberriver256/mcp-server-azure-devops"],
    "env": {
      "AZURE_DEVOPS_ORG": "${AZURE_DEVOPS_ORG}",
      "AZURE_DEVOPS_PAT": "${AZURE_DEVOPS_PAT}"
    }
  }
}

Environment Variables

AZURE_DEVOPS_ORG=your-organization-name
AZURE_DEVOPS_PAT=your-personal-access-token

Personal Access Token Setup

Create PAT

1. Go to Azure DevOps > User Settings (top-right icon) 2. Click Personal access tokens 3. Click + New Token 4. Configure: - Name: mcp-server-access - Expiration: Choose duration - Scopes: Select required permissions

Required Scopes

ScopePermission
CodeRead
BuildRead
ReleaseRead
Work ItemsRead
Project and TeamRead

Full Access (Development)

For development, you can select Full access, but limit scopes for production.

Available Tools

Projects

list_projects List all projects in the organization.

get_project Get details of a specific project.

Repositories

list_repositories List repos in a project.

get_repository Get repo details.

get_file_content Read file from repository.

Pipelines

list_pipelines List build/release pipelines.

get_pipeline Get pipeline details.

list_pipeline_runs List recent pipeline runs.

get_pipeline_run Get specific run details.

Work Items

list_work_items Query work items.

get_work_item Get work item details.

Troubleshooting

IssueSolution
"Unauthorized"Check PAT is valid and not expired
"Organization not found"Verify AZURE_DEVOPS_ORG spelling
"Project not found"Check project name/permissions
"PAT expired"Generate new PAT

Test PAT

curl -u ":$AZURE_DEVOPS_PAT" \
  "https://dev.azure.com/$AZURE_DEVOPS_ORG/_apis/projects?api-version=7.0"

Security Notes

  • Use minimum required scopes
  • Set PAT expiration (recommend 90 days max)
  • Rotate PATs regularly
  • Store PAT in environment variable, not config file
  • Never commit PAT to source control
