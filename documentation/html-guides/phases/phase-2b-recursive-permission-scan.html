Recursive Permission Scanner Flow Guide

Power Automate flow to scan and reset broken permissions in SharePoint

Overview

This guide walks you through creating a Power Automate flow that scans SharePoint folders and identifies those with broken permissions (unique role assignments).

Why Two-Step Approach?

Important Discovery:

The HasUniqueRoleAssignments property cannot be used in SharePoint REST API $filter clause. It's a computed property, not a queryable column.

Error you'll get if you try:

Column 'HasUniqueRoleAssignments' does not exist.

Solution: We must use a two-step approach:

  1. Get ALL folders under the specified path
  2. Check EACH folder's permissions individually via a separate API call
  3. Optionally reset inheritance on folders with broken permissions

Flow Structure

TRIGGER (Manual Button) │ Inputs: SiteURL, LibraryName, BaseLibraryPath, MinimumLevel, DryRun │ ├── PART A: Initialize Variables │ ├── varBaseDepth (integer) │ ├── varFoldersWithBrokenPerms (array) │ └── varProcessedCount (integer) │ ├── PART B: Get All Folders Under Path │ GET _api/web/lists/getbytitle('{lib}')/items │ $filter=FileSystemObjectType eq 1 and startswith(FileRef,'{path}') │ ├── PART C: Parse JSON Response │ ├── PART D: DEBUG - Show All Folders Found │ ├── PART E: Apply_to_each Folder │ │ │ ├── E1: HTTP - Check HasUniqueRoleAssignments │ │ │ ├── E2: Condition - If HasUniqueRoleAssignments eq true │ │ │ │ │ └── If YES: │ │ ├── E3: DEBUG - Folder Has Broken Permissions │ │ ├── E4: Append to varFoldersWithBrokenPerms │ │ └── E5: (If DryRun=false) Reset Inheritance │ │ │ └── E6: Increment varProcessedCount │ ├── PART F: DEBUG - Summary │ └── PART G: Response (Optional)

Implementation Guide

PART A: Trigger & Variables

Create the Flow

  1. Go to make.powerautomate.com
  2. Click + Create > Instant cloud flow
  3. Name: Recursive Permission Scanner
  4. Trigger: Manually trigger a flow
  5. Click Create

Add Input Parameters

Input Name Type Example
SiteURL Text https://tenant.sharepoint.com/sites/yoursite
LibraryName Text Documents
BaseLibraryPath Text /sites/yoursite/Shared Documents
MinimumLevel Text -1
DryRun Text true or false

Initialize Variables

Variable 1: varBaseDepth

  • Name: varBaseDepth
  • Type: Integer
  • Value:
length(split(triggerBody()['text_2'], '/'))

Variable 2: varFoldersWithBrokenPerms

  • Name: varFoldersWithBrokenPerms
  • Type: Array
  • Value: []

Variable 3: varProcessedCount

  • Name: varProcessedCount
  • Type: Integer
  • Value: 0

PART B: Get All Folders Under Path

Add action: Send an HTTP request to SharePoint

Site Address:

@{triggerBody()['text']}

Method: GET

Uri: (Use Expression)

concat('_api/web/lists/getbytitle(''', triggerBody()['text_1'], ''')/items?$select=Id,Title,FileLeafRef,FileRef,FileDirRef&$filter=FileSystemObjectType eq 1 and startswith(FileRef,''', triggerBody()['text_2'], ''')')

Headers:

{
  "Accept": "application/json;odata=verbose",
  "Content-Type": "application/json"
}

Rename Action: HTTP_Get_All_Folders

Note: We do NOT filter by HasUniqueRoleAssignments here - that's checked in the loop!

PART C: Parse JSON Response

Add action: Parse JSON

Content:

body('HTTP_Get_All_Folders')

Schema:

{
  "type": "object",
  "properties": {
    "d": {
      "type": "object",
      "properties": {
        "results": {
          "type": "array",
          "items": {
            "type": "object",
            "properties": {
              "Id": { "type": "integer" },
              "Title": { "type": ["string", "null"] },
              "FileLeafRef": { "type": "string" },
              "FileRef": { "type": "string" },
              "FileDirRef": { "type": "string" }
            }
          }
        }
      }
    }
  }
}

Rename Action: Parse_JSON_All_Folders

PART D: DEBUG - Show All Folders Found

Add action: Compose

Inputs:

json(concat('{"Step":"All Folders Found","TotalFolders":', length(body('Parse_JSON_All_Folders')?['d']?['results']), ',"SiteURL":"', triggerBody()['text'], '","BaseLibraryPath":"', triggerBody()['text_2'], '"}'))

Rename Action: DEBUG_AllFoldersFound

PART E: Apply_to_each Folder (The Loop)

Add action: Apply to each

Select an output from previous steps:

body('Parse_JSON_All_Folders')?['d']?['results']

Rename Action: Apply_to_each_Folder

E1: HTTP - Check HasUniqueRoleAssignments

Inside the loop, add: Send an HTTP request to SharePoint

Site Address: @{triggerBody()['text']}

Method: GET

Uri:

concat('_api/web/GetFolderByServerRelativeUrl(''', items('Apply_to_each_Folder')?['FileRef'], ''')/ListItemAllFields?$select=HasUniqueRoleAssignments')

Headers:

{
  "Accept": "application/json;odata=verbose"
}

Rename Action: HTTP_Check_Permissions

E2: Condition - Check If HasUniqueRoleAssignments is True

Add action: Condition

Condition:

body('HTTP_Check_Permissions')?['d']?['HasUniqueRoleAssignments']

is equal to true

Rename Action: Condition_HasBrokenPerms

Inside "If yes" branch:

E3: DEBUG - Folder Has Broken Permissions

Add: Compose

json(concat('{"Step":"BROKEN PERMISSIONS FOUND","FolderName":"', items('Apply_to_each_Folder')?['FileLeafRef'], '","FolderPath":"', items('Apply_to_each_Folder')?['FileRef'], '"}'))

Rename Action: DEBUG_BrokenPermission

E4: Append to varFoldersWithBrokenPerms

Add: Append to array variable

  • Name: varFoldersWithBrokenPerms
  • Value:
{
  "FolderName": "@{items('Apply_to_each_Folder')?['FileLeafRef']}",
  "FolderPath": "@{items('Apply_to_each_Folder')?['FileRef']}"
}
E5: (Optional) Reset Inheritance

Add another Condition to check DryRun:

triggerBody()['text_4'] is equal to false

Inside "If yes" (DryRun is false):

Add: Send an HTTP request to SharePoint

Method: POST

Uri:

concat('_api/web/GetFolderByServerRelativeUrl(''', items('Apply_to_each_Folder')?['FileRef'], ''')/ListItemAllFields/ResetRoleInheritance()')
Warning: ResetRoleInheritance() is destructive - it removes all custom permissions permanently!

E6: Increment varProcessedCount

Outside the "If yes" branch, still inside Apply_to_each:

Add: Increment variable

  • Name: varProcessedCount
  • Value: 1

PART F: DEBUG - Summary

After the Apply_to_each loop, add: Compose

Inputs:

json(concat('{"Step":"SCAN COMPLETED","TotalFoldersScanned":', variables('varProcessedCount'), ',"FoldersWithBrokenPermissions":', length(variables('varFoldersWithBrokenPerms')), ',"DryRun":"', triggerBody()['text_4'], '","CompletedAt":"', utcNow(), '"}'))

Rename Action: DEBUG_Summary

PART G: Response (Optional)

Add: Response (Premium connector)

Status Code: 200

Body:

{
  "status": "completed",
  "totalScanned": @{variables('varProcessedCount')},
  "brokenPermissionsCount": @{length(variables('varFoldersWithBrokenPerms'))},
  "folders": @{variables('varFoldersWithBrokenPerms')}
}

API Reference

Get All Folders

GET _api/web/lists/getbytitle('{LibraryName}')/items
  ?$select=Id,Title,FileLeafRef,FileRef,FileDirRef
  &$filter=FileSystemObjectType eq 1 and startswith(FileRef,'{BasePath}')

Check Folder Permissions

GET _api/web/GetFolderByServerRelativeUrl('{FileRef}')/ListItemAllFields
  ?$select=HasUniqueRoleAssignments

Reset Inheritance

POST _api/web/GetFolderByServerRelativeUrl('{FileRef}')/ListItemAllFields/ResetRoleInheritance()

Troubleshooting

Error: BadGateway (502)

Cause: SharePoint throttling

Solution: Add delays between API calls, configure retry policy, reduce batch size

Error: Column does not exist

Cause: Using HasUniqueRoleAssignments in $filter

Solution: Use this two-step approach instead!

Error: List does not exist

Cause: Wrong library name

Solution: Use internal name (usually Documents not Shared Documents)
