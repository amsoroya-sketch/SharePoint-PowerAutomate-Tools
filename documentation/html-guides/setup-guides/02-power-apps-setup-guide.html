Power Apps Setup Guide

Complete guide for developing Canvas and Model-driven apps.

Access Power Apps

Web Interface

https://make.powerapps.com

Through Microsoft 365

1. Go to https://office.com 2. Click App Launcher (9 dots) 3. Select Power Apps

App Types

TypeBest ForData Source
CanvasCustom UI, mobile-firstAny connector
Model-drivenData-centric, business appsDataverse only
PortalExternal-facing websitesDataverse

Canvas Apps

Create Canvas App

1. Click + Create > Blank app 2. Select Blank canvas app 3. Choose format (Tablet/Phone) 4. Name your app 5. Click Create

App Structure

App
├── Screens
│   ├── Screen1
│   │   ├── Controls (buttons, labels, galleries)
│   │   └── Components
│   └── Screen2
├── Data Sources
├── Media
└── Components

Common Controls

ControlPurpose
LabelDisplay text
ButtonTrigger actions
TextInputUser input
GalleryDisplay list of items
FormEdit/view records
DataTableTabular data display

Key Formulas

// Navigate between screens
Navigate(Screen2, ScreenTransition.Fade)




// Filter data
Filter(DataSource, Status = "Active")




// Lookup single record
LookUp(Users, ID = Gallery1.Selected.UserID)




// Collect data
Collect(LocalCollection, {Name: "Item1", Value: 100})




// Update record
Patch(DataSource, LookUp(DataSource, ID = 1), {Status: "Complete"})




// Refresh data
Refresh(DataSource)

Connect to Dataverse

1. Click Data in left panel 2. Click + Add data 3. Search for your table 4. Select and connect

Connect to SharePoint

1. Click Data > + Add data 2. Search "SharePoint" 3. Enter site URL 4. Select list/library

Model-driven Apps

Create Model-driven App

1. Click + Create > Blank app 2. Select Blank app based on Dataverse 3. Name your app 4. Click Create

App Designer

App
├── Navigation (Site Map)
│   ├── Area 1
│   │   ├── Group 1
│   │   │   ├── Page (Table View)
│   │   │   └── Page (Dashboard)
│   │   └── Group 2
│   └── Area 2
├── Pages
│   ├── Table Views
│   ├── Dashboards
│   └── Custom Pages
└── Components

Add Table to App

1. Open App Designer 2. Click + Add page 3. Select Dataverse table 4. Choose table 5. Select views and forms 6. Click Add

Customize Forms

1. Go to table in Power Apps 2. Click Forms 3. Select form to edit 4. Drag/drop columns 5. Add sections/tabs 6. Save and Publish

Customize Views

1. Go to table 2. Click Views 3. Select or create view 4. Add/remove columns 5. Set filters and sorting 6. Save and Publish

Solutions

Create Solution

1. Click Solutions in left nav 2. Click + New solution 3. Enter: - Display name - Name (no spaces) - Publisher - Version 4. Click Create

Add Components to Solution

1. Open solution 2. Click + Add existing 3. Select component type (App, Table, Flow) 4. Choose components 5. Click Add

Export Solution

1. Open solution 2. Click Export 3. Choose: - Unmanaged: For development - Managed: For production 4. Click Export 5. Download zip file

Import Solution

1. Click Import solution 2. Browse to zip file 3. Click Next 4. Review and click Import

CLI Commands (PAC)

List Canvas Apps

pac canvas list

Download Canvas App

pac canvas download --canvas-name "AppName" --file app.msapp

Upload Canvas App

pac canvas upload --file app.msapp --canvas-name "AppName"

Solution Commands

# Export solution with apps
pac solution export --name SolutionName --path solution.zip



Import solution

pac solution import --path solution.zip

Best Practices

Canvas Apps

  • Use components for reusable UI
  • Minimize data calls (use collections)
  • Delegate filtering to data source
  • Test on target devices

Model-driven Apps

  • Use business rules for validation
  • Configure security roles
  • Use views for different scenarios
  • Leverage business process flows

General

  • Use solutions for ALM
  • Version your apps
  • Test in development first
  • Document customizations
