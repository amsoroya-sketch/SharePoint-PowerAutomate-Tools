Power BI Setup Guide

Complete guide for creating reports and dashboards with Power BI.

Access Power BI

Web Interface

https://app.powerbi.com

Desktop App

Download: https://powerbi.microsoft.com/desktop/

Through Microsoft 365

1. Go to https://office.com 2. Click App Launcher (9 dots) 3. Select Power BI

Connect to Dataverse

From Power BI Desktop

1. Click Get Data > Power Platform > Dataverse 2. Enter environment URL: https://[org].crm.dynamics.com 3. Sign in with credentials 4. Select tables: - sp_scansession - sp_folderrecord - sp_permissionentry 5. Click Load

Connection String

Environment URL: https://[org].crm.dynamics.com
Authentication: Organizational Account

Connect to SharePoint

From Power BI Desktop

1. Click Get Data > Online Services > SharePoint Online List 2. Enter site URL: https://tenant.sharepoint.com/sites/sitename 3. Select lists/libraries 4. Choose Load or Transform Data

Project Reports

Scan Summary Dashboard

Visuals:
├── Card: Total Scans
├── Card: Total Folders Scanned
├── Card: Folders with Unique Permissions
├── Line Chart: Scans Over Time
├── Bar Chart: Folders per Scan
└── Table: Recent Scan Details

Permission Analysis Report

Visuals:
├── Pie Chart: Permission Types
├── Bar Chart: Folders by Depth
├── Matrix: Permissions by User/Group
└── Drill-through: Folder Details

DAX Measures

Common Measures

// Total Scans
Total Scans = COUNTROWS(sp_scansession)




// Total Folders
Total Folders = SUM(sp_scansession[sp_totalfolders])




// Folders with Unique Permissions
Unique Permission Folders = SUM(sp_scansession[sp_folderswithuniqueperms])




// Percentage with Unique Permissions
Unique Permission % =
DIVIDE(
    [Unique Permission Folders],
    [Total Folders],
    0
) * 100




// Average Folders per Scan
Avg Folders per Scan = AVERAGE(sp_scansession[sp_totalfolders])




// Success Rate
Success Rate =
DIVIDE(
    COUNTROWS(FILTER(sp_scansession, sp_scansession[sp_status] = "Completed")),
    [Total Scans],
    0
) * 100

Publish to Service

Publish from Desktop

1. Click Publish in ribbon 2. Select workspace 3. Click Select 4. Open in Power BI Service

Configure Refresh

1. Open dataset settings 2. Click Scheduled refresh 3. Configure refresh schedule 4. Set credentials for data sources

Embed in Power Apps

Get Embed URL

1. Open report in Power BI Service 2. Click File > Embed report > Website or portal 3. Copy embed URL

Add to Canvas App

1. Insert Power BI tile control 2. Set TileUrl property to embed URL

Add to Model-driven App

1. Create custom page 2. Add Power BI component 3. Configure workspace and report

CLI Commands

Power BI CLI (via Azure CLI)

# Install extension
az extension add --name powerbicli



Login

az login



List workspaces

az powerbi workspace list



List reports in workspace

az powerbi report list --workspace-name "WorkspaceName"

REST API

Base URL

https://api.powerbi.com/v1.0/myorg/

Common Endpoints

# Get workspaces
GET /groups



Get reports in workspace

GET /groups/{workspaceId}/reports



Get datasets

GET /groups/{workspaceId}/datasets



Refresh dataset

POST /groups/{workspaceId}/datasets/{datasetId}/refreshes

Using Azure CLI

TOKEN=$(az account get-access-token --resource https://analysis.windows.net/powerbi/api --query accessToken -o tsv)




curl -X GET "https://api.powerbi.com/v1.0/myorg/groups" \
  -H "Authorization: Bearer $TOKEN"
