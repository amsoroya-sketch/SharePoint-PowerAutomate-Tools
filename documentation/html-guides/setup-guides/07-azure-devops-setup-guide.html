Azure DevOps Setup Guide

Complete guide for CI/CD pipelines and ALM with Azure DevOps.

Access Azure DevOps

URL

https://dev.azure.com/[organization]

Create Organization

1. Go to https://dev.azure.com 2. Click Start free 3. Sign in with Microsoft account 4. Create organization name

Project Setup

Create Project

1. Click + New project 2. Configure: - Project name - Description - Visibility (Private/Public) - Version control (Git) 3. Click Create

Project Structure

Project
├── Repos (Git repositories)
├── Pipelines (CI/CD)
├── Boards (Work items)
├── Test Plans
└── Artifacts

Repository Setup

Create Repo

1. Click Repos > Files 2. Click + New repository 3. Enter name 4. Add README/gitignore 5. Click Create

Clone Repo

git clone https://dev.azure.com/[org]/[project]/_git/[repo]

Push Existing Code

git remote add origin https://dev.azure.com/[org]/[project]/_git/[repo]
git push -u origin main

Power Platform ALM Pipeline

Pipeline for Solution Export/Import

# azure-pipelines.yml
trigger:
  - main




pool:
  vmImage: 'windows-latest'




variables:
  SolutionName: 'SharePointPermissionScanner'
  SourceEnvironment: 'https://[org]-dev.crm.dynamics.com'
  TargetEnvironment: 'https://[org]-prod.crm.dynamics.com'




stages:
  - stage: Export
    displayName: 'Export Solution'
    jobs:
      - job: ExportSolution
        steps:
          - task: PowerPlatformToolInstaller@2
            displayName: 'Install Power Platform CLI'




          - task: PowerPlatformExportSolution@2
            displayName: 'Export Solution'
            inputs:
              authenticationType: 'PowerPlatformSPN'
              PowerPlatformSPN: 'DevEnvironment'
              SolutionName: '$(SolutionName)'
              SolutionOutputFile: '$(Build.ArtifactStagingDirectory)/$(SolutionName).zip'




          - task: PublishBuildArtifacts@1
            displayName: 'Publish Artifact'
            inputs:
              PathtoPublish: '$(Build.ArtifactStagingDirectory)'
              ArtifactName: 'solution'




  - stage: Deploy
    displayName: 'Deploy to Production'
    dependsOn: Export
    condition: succeeded()
    jobs:
      - deployment: DeploySolution
        environment: 'Production'
        strategy:
          runOnce:
            deploy:
              steps:
                - task: PowerPlatformToolInstaller@2
                  displayName: 'Install Power Platform CLI'




                - task: PowerPlatformImportSolution@2
                  displayName: 'Import Solution'
                  inputs:
                    authenticationType: 'PowerPlatformSPN'
                    PowerPlatformSPN: 'ProdEnvironment'
                    SolutionInputFile: '$(Pipeline.Workspace)/solution/$(SolutionName).zip'

Service Connections

Create Power Platform Connection

1. Go to Project Settings > Service connections 2. Click New service connection 3. Select Power Platform 4. Configure: - Server URL: https://[org].crm.dynamics.com - Tenant ID - Application ID - Client Secret 5. Click Save

Required App Permissions

Dynamics CRM:
└── user_impersonation (Delegated)

CLI Commands

Install Extension

az extension add --name azure-devops

Configure Defaults

az devops configure --defaults organization=https://dev.azure.com/myorg project=MyProject

Login

az devops login

Or use PAT

echo $PAT | az devops login --organization https://dev.azure.com/myorg

Pipelines

# List pipelines
az pipelines list --output table



Run pipeline

az pipelines run --name "Pipeline Name"



Get run details

az pipelines runs show --id RUN_ID



List runs

az pipelines runs list --pipeline-id PIPELINE_ID

Repos

# List repos
az repos list --output table



Create repo

az repos create --name "RepoName"



Create PR

az repos pr create --title "PR Title" --source-branch feature --target-branch main

Branch Strategy

main (production)
├── release/v1.0
├── develop
│   ├── feature/add-permission-scan
│   ├── feature/improve-reporting
│   └── bugfix/fix-column-name

Branch Policies

1. Go to Repos > Branches 2. Click ... on branch > Branch policies 3. Configure: - Require PR - Require reviewers - Build validation - Comment resolution

Best Practices

Pipelines

  • Use YAML pipelines (version controlled)
  • Implement approval gates
  • Use variable groups for secrets
  • Run solution checker before deploy

Solutions

  • Use managed solutions in production
  • Version solutions properly
  • Document dependencies
  • Test in lower environments first

Security

  • Use service principals (not user accounts)
  • Rotate secrets regularly
  • Limit pipeline permissions
  • Audit deployments
