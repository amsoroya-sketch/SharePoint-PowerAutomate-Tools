SharePoint Online Setup Guide

Complete guide for SharePoint Online development and administration.

Access SharePoint

Direct URL

https://[tenant].sharepoint.com

Project Site

https://abctest179.sharepoint.com/sites/Permission-Scanner-Test

Through Microsoft 365

1. Go to https://office.com 2. Click App Launcher (9 dots) 3. Select SharePoint

URL Structure

https://[tenant].sharepoint.com/sites/[sitename]/[library]/[folder]




Components:
├── tenant: Your organization (abctest179)
├── /sites/: Site collection path
├── sitename: Site name (Permission-Scanner-Test)
├── library: Document library (TestLibrary_Basic)
└── folder: Folder path

Site Management

Create Team Site

1. Click + Create site 2. Select Team site 3. Configure: - Site name - Privacy settings - Members 4. Click Create

Create Communication Site

1. Click + Create site 2. Select Communication site 3. Choose design 4. Configure settings 5. Click Create

Document Libraries

Create Library

1. Go to site 2. Click + New > Document library 3. Enter name 4. Click Create

Library Settings

1. Click Settings (gear icon) 2. Select Library settings 3. Configure: - Versioning - Permissions - Columns - Views

Folders

Create Folder

1. Open document library 2. Click + New > Folder 3. Enter folder name 4. Click Create

Navigate Folders

  • Click folder name to enter
  • Use breadcrumb to navigate up
  • Click library name to return to root

Permissions

Permission Levels

LevelPermissions
Full ControlAll permissions
DesignView, add, edit, delete, approve
EditView, add, edit, delete
ContributeView, add, edit
ReadView only
View OnlyView without download

Check Permissions

1. Select item 2. Click ... > Manage access 3. View current permissions

Break Inheritance

1. Select folder 2. Click Manage access 3. Click Advanced (or ⚙️) 4. Click Stop Inheriting Permissions 5. Confirm action

Grant Permission

1. Click Manage access 2. Click Grant access 3. Enter user/group 4. Select permission level 5. Click Grant

Remove Permission

1. Click Manage access 2. Find user/group 3. Click Remove

Reset Inheritance

1. Go to folder permissions 2. Click Delete unique permissions 3. Confirm to inherit from parent

REST API

Base URL

https://[tenant].sharepoint.com/sites/[sitename]/_api/

Common Endpoints

Get Web Info

GET _api/web

Get Lists

GET _api/web/lists

Get Folder Contents

GET _api/web/GetFolderByServerRelativeUrl('/sites/sitename/Shared Documents')/Folders

Get Folder Permissions

GET _api/web/GetFolderByServerRelativeUrl('/path')/ListItemAllFields/RoleAssignments

Check Unique Permissions

GET _api/web/GetFolderByServerRelativeUrl('/path')/ListItemAllFields/HasUniqueRoleAssignments

Headers

{
  "Accept": "application/json;odata=nometadata",
  "Content-Type": "application/json;odata=verbose"
}

Power Automate Integration

Send HTTP Request to SharePoint

Action: Send an HTTP request to SharePoint
Site Address: https://tenant.sharepoint.com/sites/sitename
Method: GET
Uri: _api/web/GetFolderByServerRelativeUrl('/sites/sitename/Shared Documents')/Folders
Headers:
  Accept: application/json;odata=nometadata

Parse Response

@body('Send_HTTP_request_to_SharePoint')?['d']?['results']

CLI Commands

m365 CLI

# Get site info
m365 spo site get --url https://tenant.sharepoint.com/sites/sitename



List document libraries

m365 spo list list --webUrl https://tenant.sharepoint.com/sites/sitename



List folder contents

m365 spo folder list --webUrl https://tenant.sharepoint.com/sites/sitename --folderUrl "/Shared Documents"

PnP PowerShell

# Connect
Connect-PnPOnline -Url "https://tenant.sharepoint.com/sites/sitename" -Interactive



Get folders

Get-PnPFolderItem -FolderSiteRelativeUrl "Shared Documents"



Check permissions

Get-PnPListItemPermission -List "Documents" -Identity 1

Troubleshooting

IssueSolution
"Access denied"Check permissions
"Site not found"Verify URL
"List not found"Check library name
"Folder not found"Check path is correct

Verify Permissions

1. Go to site settings 2. Click Site permissions 3. Check user access level

Check API Response

1. Navigate to _api/web in browser 2. Review JSON response 3. Look for error messages

Best Practices

Permissions

  • Use groups, not individuals
  • Minimize custom permissions
  • Document permission changes
  • Audit regularly

Libraries

  • Use meaningful names
  • Set up versioning
  • Configure content types
  • Create useful views

Performance

  • Avoid deeply nested folders (>5 levels)
  • Use indexed columns for large libraries
  • Limit items per view
