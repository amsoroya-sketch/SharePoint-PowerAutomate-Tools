Naming Convention

  • Publisher prefix: sp_ (SharePoint Scanner)
  • Use lowercase with underscores
  • Example: sp_scansession, sp_folderrecord

Create Custom Table

Steps

1. Go to Tables > + New table 2. Configure: - Display name - Plural name - Schema name (auto-generated) - Primary column name 3. Click Save

Project Tables

Scan Sessions (sp_scansession)

Purpose: Track each scan execution




Columns:
├── sp_scansessionid (Primary, GUID)
├── sp_name (Text) - Session name
├── sp_siteurl (Text) - SharePoint site URL
├── sp_libraryname (Text) - Document library
├── sp_status (Choice) - Running/Completed/Failed
├── sp_totalfolders (Whole Number)
├── sp_folderswithuniqueperms (Whole Number)
├── sp_startedon (Date/Time)
├── sp_completedon (Date/Time)
├── sp_errormessage (Multi-line Text)
└── sp_haserror (Yes/No)

Folder Records (sp_folderrecord)

Purpose: Store folders with unique permissions




Columns:
├── sp_folderrecordid (Primary, GUID)
├── sp_name (Text) - Folder name
├── sp_folderpath (Text) - Full path
├── sp_scansession (Lookup) - Parent session
├── sp_depth (Whole Number) - Nesting level
├── sp_hasuniquepermissions (Yes/No)
└── sp_permissioncount (Whole Number)

Permission Entries (sp_permissionentry)

Purpose: Store permission details




Columns:
├── sp_permissionentryid (Primary, GUID)
├── sp_folderrecord (Lookup) - Parent folder
├── sp_principaltype (Choice) - User/Group
├── sp_principalname (Text) - User or group name
├── sp_permissionlevel (Text) - Read/Edit/Full Control
└── sp_isinherited (Yes/No)

Query Data (OData)

Filter Syntax

// Equals
$filter=sp_status eq 'Completed'




// Not equals
$filter=sp_status ne 'Failed'




// Contains
$filter=contains(sp_name, 'Test')




// Date comparison
$filter=sp_createdon gt 2026-01-01




// Multiple conditions
$filter=sp_status eq 'Completed' and sp_totalfolders gt 0

Select Columns

$select=sp_name,sp_status,sp_totalfolders

Order By

$orderby=sp_createdon desc

Expand Related Records

$expand=sp_scansession($select=sp_name)

Action: Update a row
Table name: Scan Sessions

Columns:
  sp_status: Completed
  sp_completedon: @{utcNow()}

Table name: Folder Records
Filter rows: sp_scansession/sp_scansessionid eq '@{variables('SessionId')}'

Best Practices

Design

  • Use consistent naming convention
  • Add descriptions to all tables/columns
  • Create meaningful primary column names
  • Use Choice columns for fixed options

Performance

  • Index frequently queried columns
  • Use views with filters
  • Limit returned columns with $select
  • Paginate large result sets

Maintenance

  • Document table purposes
  • Version control schema changes
  • Test in development environment first
