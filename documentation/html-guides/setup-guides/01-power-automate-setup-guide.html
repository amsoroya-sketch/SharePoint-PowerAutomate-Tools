Power Automate Setup Guide

Complete guide for developing and deploying Power Automate cloud flows.

Access Power Automate

Web Interface

https://make.powerautomate.com

Through Microsoft 365

1. Go to https://office.com 2. Click App Launcher (9 dots) 3. Select Power Automate

Through Power Apps

1. Go to https://make.powerapps.com 2. Click Flows in left navigation

Environment Setup

Select Environment

1. Click environment selector (top-right) 2. Choose your target environment 3. Verify with environment name displayed

Project Environment

Create Cloud Flow

From Blank

1. Click + Create > Instant cloud flow 2. Name your flow 3. Select trigger type 4. Click Create

From Template

1. Click Templates 2. Search for template 3. Click template to use 4. Customize as needed

Trigger Types

TypeUse Case
Manual triggerButton/on-demand flows
AutomatedEvent-driven (SharePoint, Dataverse)
ScheduledTimer-based execution
InstantTriggered from apps/buttons

Flow Structure

Actions

  • Standard actions: Compose, Condition, Apply to each, etc.
  • Connectors: SharePoint, Dataverse, HTTP, etc.
  • Custom connectors: External APIs

Control Flow

┌─────────────────────────────────────┐
│           Trigger                    │
├─────────────────────────────────────┤
│         Initialize Variables         │
├─────────────────────────────────────┤
│         Try Scope                    │
│   ├── Main Logic                    │
│   └── Business Operations            │
├─────────────────────────────────────┤
│         Catch Scope                  │
│   └── Error Handling                 │
├─────────────────────────────────────┤
│         Finally Scope                │
│   └── Cleanup / Update Status        │
└─────────────────────────────────────┘

Expression Reference

Trigger Functions

FunctionReturns
trigger()Trigger metadata (status, timing)
triggerBody()Input values from trigger
triggerOutputs()Headers + body

Example

// Get user input
triggerBody()['text']




// Get trigger status
trigger()['status']

Error Handling

Try-Catch-Finally Pattern

1. Try Scope: Main business logic 2. Catch Scope: Error handling (Configure Run After = "has failed", "has timed out") 3. Finally Scope: Always runs (Configure Run After = all conditions)

Configure Run After

1. Click ... on action 2. Select Configure run after 3. Check desired conditions: - is successful - has failed - is skipped - has timed out

Deployment

Export Flow (via Solution)

1. Go to Solutions 2. Open your solution 3. Click Export 4. Choose Unmanaged/Managed 5. Download zip file

Import Flow

1. Go to Solutions 2. Click Import 3. Browse to solution zip 4. Follow import wizard

CLI Deployment

# Export
pac solution export --name SolutionName --path solution.zip



Import

pac solution import --path solution.zip

Testing

Test Flow

1. Click Test (top-right) 2. Select Manually 3. Click Test 4. Provide input values 5. Click Run flow

View Run History

1. Open flow 2. Scroll to 28-day run history 3. Click on a run to see details

Check Action Outputs

1. Click on completed action 2. Expand Inputs and Outputs 3. Review data

Best Practices

Naming Conventions

  • Use PascalCase for actions: Get_Folder_Items
  • Use descriptive names
  • Include scope purpose: Try_Scope, Catch_Scope

Performance

  • Use Select action to reduce data
  • Filter early in the flow
  • Use pagination for large datasets

Maintenance

  • Add comments in Compose actions
  • Use Try-Catch-Finally pattern
  • Log important events to Dataverse
